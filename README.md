## 学習ログ
- 2025-12-26：OSI参照モデル＝通信を分解する設計思想だと理解

```text
L7 アプリケーション層 : HTTP / Web / アプリのルール
L6 プレゼンテーション層 : 文字コード / 暗号化
L5 セッション層 : 接続の開始・維持
L4 トランスポート層 : TCP / UDP（信頼性）
L3 ネットワーク層 : IP / ルーティング
L2 データリンク層 : MAC / 同一LAN
L1 物理層 : 電気 / ケーブル
```

L4 トランスポート層
- TCP : 信頼性あり（順序・再送・制御）
- UDP : 速い・軽い（信頼性なし）

👉 上＝人寄り / 下＝機械寄り

- OSIは「通信を7段に分けて考える設計思想」

- インフラは「暗記」じゃなく「包んで渡す設計」
- カプセル化＝データに役割ごとの情報を重ねること

アプリのデータ
↓
L4（TCP/UDP） : ポート番号・信頼性の情報を付ける
↓
L3（IP） : 送信元IP / 宛先IPを付ける
↓
L2（MAC） : 宛先MACを付ける
↓
L1（物理） : 電気信号として送る
